실행 방법 및 버전 정합성 가이드

1) Frontend (Sprint/)
- 런타임: Node.js (Vite 7 기준 Node 18+ 권장, @types/node 25 기준 Node 22 사용 권장)
- 패키지 매니저: npm (package-lock.json 기준)
- 설치: cd Sprint && npm install
- 개발 서버: npm run dev
- 빌드: npm run build
- 프리뷰: npm run preview

2) Backend (backend/)
- 런타임: Node.js (Frontend와 동일 버전 권장)
- 패키지 매니저: npm (package-lock.json 기준)
- 설치: cd backend && npm install
- 환경변수: DATABASE_URL 필요 (Prisma/MySQL 연결)
- 개발 서버: npm run dev
- 빌드/실행: npm run build && npm run start
- 참고: 스키마 변경 시 Prisma generate/migrate 필요할 수 있음

3) Resume Screening API (resume_screening/)
- 런타임: Python 3.9 (conda 권장)
- 설치(Conda): CONDA_SOLVER=classic conda create -n resume-screening python=3.9 -c conda-forge -c defaults -y
- 설치(Pip): conda activate resume-screening && pip install -r requirements.txt
- Ollama: Ollama 앱 실행 필요(로컬 11434 포트). `ollama pull llama3`로 모델 다운로드 후 사용.
- 환경변수: OLLAMA_BASE_URL(기본 http://127.0.0.1:11434), OLLAMA_MODEL(예: llama3:latest)
- 실행: conda activate resume-screening && python main.py

버전 정합성 체크
- Node.js: Frontend/Backend 동일 메이저 버전 사용 권장. @types/node 25 기준 Node 22 권장, Vite 7 사용 시 최소 Node 18 필요.
- TypeScript: Frontend와 Backend 모두 5.9.x 사용 중. 가능하면 동일 마이너 유지.
- Prisma: prisma와 @prisma/client 버전 동일(6.19.1) 유지 필요.
- DB: Prisma datasource는 MySQL로 설정됨(DATABASE_URL 확인).
- Python: resume_screening은 Python 3.9 고정.
- Ollama: resume_screening은 Ollama 서버가 필요. 모델명은 `ollama list`로 확인 후 `OLLAMA_MODEL`에 맞춤.
